Getting Started

Setup

For setting up, it is recommended to first read the Tools page then come back to this page.

Typical workflow

This section will describe a typical workflow from writing a new piece of code, to checking it agianst formats and testing it, to seeing code coverage and documentation for it. To understand what each script does, it is recommended to open them up in a text editor and seeing what they do.

  1. Write a new header file template

  2. Write a new cpp file for it

  3. Write a test for it to test that the new functions work

  4. Run ./scripts/build/tests.sh to build the testing executable

  5. If clang-tidy or clang-format say that there is some misformatted code or if there are errors in the build, then fix them and run again.

    1. At this point you may want to consider running ./scripts/standalone/apply_clang_format.py if you agree with the changes proposed by clang-format

  6. Run ./scripts/standalone/test.sh to run the tests and generate the code coverage report which can be found by opening docs/index.html in a browser and choose the Code Coverage option

  7. Integrate the new functions into the program and test again

  8. Once you have a main executable ready to be run you can run ./scripts/build/debug.sh to generate a debug build of your code.

  9. You can run your code by running ./build/bin/main + arguments, which is the main executable generated

  10. Here you may update the documentation. You can run ./scripts/build/docs.sh, which will generate the documentation in the docs folder. Similarly with the code coverage, you may open docs/index.html in a browser and this time choose the Documentation option.

  11. After you are satisfied and wish to generate a release executable, you may run ./scripts/build/release.sh and then you may execute it by running ./build/src/main.
